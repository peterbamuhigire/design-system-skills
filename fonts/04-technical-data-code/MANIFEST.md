# MANIFEST - fonts/04-technical-data-code

Precise, engineered type for dashboards, admin panels, API docs, data products, technical
documentation, and code-adjacent interfaces. Binaries are gitignored; drop files in per device.

| Family | Role | Voice | Best for | Avoid for | Pair with | Licence | Embed? | Redistribute file? | Present? |
|---|---|---|---|---|---|---|---|---|---|
| IBM Plex Sans | UI/body | Engineered, rational, readable | Dashboards, admin UI, API docs, technical SaaS | Expressive marketing heroes | IBM Plex Serif, IBM Plex Mono | OFL | Yes | Allowed (kept out by policy) | drop-in |
| IBM Plex Serif | Display/body accent | Technical-formal, editorial-engineered | Technical reports, API documentation, standards docs | Youthful product pages | IBM Plex Sans, IBM Plex Mono | OFL | Yes | Allowed (kept out by policy) | drop-in |
| IBM Plex Mono | Code/data | Technical, precise | Code blocks, IDs, logs, tabular labels | Paragraph body, long marketing copy | IBM Plex Sans | OFL | Yes | Allowed (kept out by policy) | drop-in |
| JetBrains Mono | Code/data accent | Developer-native, clear | Code editors, CLI docs, snippets, data labels | Non-technical branding | IBM Plex Sans, Public Sans | OFL | Yes | Allowed (kept out by policy) | drop-in |
| Fira Code | Code | Developer, ligature-friendly | Code blocks and examples | UI body, non-code copy | IBM Plex Sans | OFL | Yes | Allowed (kept out by policy) | drop-in |
| Space Mono | Short labels only | Retro-technical, conspicuous | Small labels, badges, sparse technical accents | Body copy, primary product display | IBM Plex Sans | OFL | Yes | Allowed (kept out by policy) | drop-in (use sparingly) |

**Baseline guarantee:** listed OFL families are available via Google Fonts or public upstreams.
Never mix monospace and proportional faces in the same role; monospace is for code/data accents.
