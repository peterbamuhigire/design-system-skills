|---|---|
| **LCP** (Largest Contentful Paint) | ≤ **2.5 s** |

## 3. 2026 trends — craft vs. slop

These signal whether work is authored with intent or generated as slop. Craft-vs-slop calls are `(inference)`, anchored to sourced critiques.

### 3.1 Variable fonts
One file packs a whole family along continuous axes (Weight, Width, Slant, Optical Size, custom) ([web.dev/articles/variable-fonts](https://web.dev/articles/variable-fonts) · [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_fonts/Variable_fonts_guide)). web.dev cites a Monotype experiment combining **12 fonts into one variable file for an 88% size reduction** ([web.dev](https://web.dev/articles/variable-fonts)). Supported across modern browsers since May 2020. **Caveat in source:** for a single weight, a many-axis file can be *heavier* than a static cut.
- **Craft `(inference)`:** optical sizing tuned display vs. body; fluid typography mapping an axis to context; full hierarchy on one budgeted file.
- **Slop `(inference)`:** a single static weight with no hierarchy, or shipping many static weight files where one variable font would do.

### 3.2 Motion / micro-interactions
Natural motion uses velocity-aware springs and custom curves; Motion (motion.dev, evolved from Framer Motion) generates CSS `linear()` curves ([motion.dev](https://motion.dev/docs/easing-functions)). **GSAP became 100% free for all uses on 30 April 2025** under Webflow ([webflow.com/blog/gsap-becomes-free](https://webflow.com/blog/gsap-becomes-free) · [gsap.com/pricing](https://gsap.com/pricing/)). `prefers-reduced-motion` must be honoured — strip non-essential movement while keeping functional feedback ([web.dev](https://web.dev/articles/prefers-reduced-motion)).
- **Craft `(inference)`:** purposeful, short, state-communicating motion with a real reduced-motion path.
- **Slop `(inference)`:** scroll-jacking, reveal-on-everything, decorative parallax, ignoring `prefers-reduced-motion`.

### 3.3 Bento-box layouts
Compartmentalised grid popularised (not invented) by Apple via WWDC keynotes — "immaculately laid-out bento boxes containing key features" ([MacStories](https://www.macstories.net/news/wwdc-2025-all-the-small-things-bento-box-version/)). Now "a dominant format for product showcases" ([Studio Meyer](https://studiomeyer.io/en/blog/bento-grid-layouts)) — but listed among "10 trends creatives are so over in 2026," the failure mode being grids "used without thought, without purpose" ([Creative Boom](https://www.creativeboom.com/insight/10-trends-creatives-are-so-over-in-2026/)).
- **Craft `(inference)`:** each cell carries a distinct purpose; size/placement encode hierarchy.
- **Cliché `(inference)`:** an interchangeable decorative box matrix with no cell logic.

### 3.4 Brutalism / anti-design
"Brutalist web design" coined by Pascal Deville (BrutalistWebsites.com) ([LogRocket](https://blog.logrocket.com/ux-design/web-brutalism/)). **Neobrutalism** (NN/g) is "high contrast, blocky layouts, bold colors, thick borders, and 'unpolished' elements" — but "without balance, it can overwhelm users and hinder accessibility" ([NN/g](https://www.nngroup.com/articles/neobrutalism/)).
- **Craft `(inference)`:** harshness is load-bearing and serves a stated stance; information stays navigable.
- **Slop `(inference)`:** ugliness with no thesis — the "without balance… hinder accessibility" tell.

### 3.5 3D / WebGL / WebGPU
**WebGPU shipped in all major browsers** (web.dev, 25 Nov 2025): Chrome/Edge v113+, Firefox v141 (Windows), Safari v26 ([web.dev/blog/webgpu-supported-major-browsers](https://web.dev/blog/webgpu-supported-major-browsers)). Three.js is the canonical JS 3D library ([threejs.org](https://threejs.org/)); React Three Fiber moving WebGPU/TSL toward first-class ([r3f docs](https://r3f.docs.pmnd.rs/)). Active Theory and Resn are the studio references.
- **Craft `(inference)`:** 3D serving narrative/product, WebGPU with WebGL2 fallback, respects performance budgets.
- **Cliché `(inference)`:** gratuitous spinning blobs/gradient spheres, heavy unthrottled WebGL, no fallback.

### 3.6 AI-native UX patterns
**Generative UI** — "a user interface dynamically generated in real time by AI to provide an experience customized to fit the user's needs and context" ([NN/g](https://www.nngroup.com/articles/generative-ui/)); in practice, connecting tool-call results to React components ([Vercel AI SDK](https://ai-sdk.dev/docs/ai-sdk-ui/generative-user-interfaces)). The 2026 shift is "from Conversational UI to Delegative UI (managing a digital workforce)" ([NN/g State of UX 2026](https://www.nngroup.com/articles/state-of-ux-2026/)). Named agentic patterns — Intent Preview, Autonomy Dial, Explainable Rationale, Confidence Signal, Action Audit & Undo, Escalation Pathway — with the thesis "trustworthiness is an output of a design process" ([Smashing, Feb 2026](https://www.smashingmagazine.com/2026/02/designing-agentic-ai-practical-ux-patterns/)).
- **Craft `(inference)`:** generative UI tied to real tool outputs; agentic interfaces with intent preview, control, and undo.
- **Slop `(inference)`:** a bolt-on chatbot on an unchanged UI; generic AI-generated layouts with no control/confidence/reversal.
- `(synthesis)` — 2026 consensus (NN/g + Smashing): AI-native UX is judged by **control, transparency, and trust**, not the presence of a chat box.

### 3.7 Accessibility-first design
**WebAIM Million 2026 (current headline):** **95.9% of home pages had detected WCAG 2 failures**; **56,114,377 errors**; **56.1 errors per page**; led by low-contrast text (83.9% of pages) and missing alt text (53.1%) ([webaim.org/projects/million](https://webaim.org/projects/million/)). *(Prior 2025 edition: 94.8% failing, 51 errors/page — [webaim.org/projects/million/2025](https://webaim.org/projects/million/2025).)* "Designing with accessibility in mind from the start is even better than reviewing a design for accessibility" ([A11Y Project](https://www.a11yproject.com/posts/starting-a-design-with-accessibility/)).
- **Craft `(inference)`:** accessibility in the first pass — semantic order, sufficient contrast, real alt text.
- **Slop `(inference)`:** shipping the ~96% default and deferring to a later "audit" or an overlay widget.

### 3.8 Performance-as-craft
Pass all three CWV at the 75th percentile on real devices (see §2.4). A **performance budget** is "a set of limits imposed on metrics that affect site performance… a point of reference for making decisions about design, technology, and adding features" ([web.dev/articles/performance-budgets-101](https://web.dev/articles/performance-budgets-101)). Fonts tie into CLS — unmanaged web-font loading shifts content; mitigate with `font-display`, size-matched fallbacks, and preload ([web.dev/articles/optimize-cls](https://web.dev/articles/optimize-cls)). This is where typography and performance meet: a variable font keeps you under a font budget while preserving range `(inference)`.
- **Craft `(inference)`:** day-one budget, preloaded/size-matched fonts, fast treated as a feature.
- **Slop `(inference)`:** no budget, unbounded third-party scripts, lab-only optimisation while field CWV stay red.

## 4. Document & proposal craft bar

World-class consultancy decks/reports and editorial documents are graded on logic structure, typographic system, and grid discipline.

### 4.1 Consultancy report / deck craft
- **The Pyramid Principle (Barbara Minto)** — developed at McKinsey; "a standard text used by all of the major consulting firms." Core devices: **SCQA** (Situation → Complication → Question → Answer) and **MECE** grouping; lead with the answer, then support top-down ([barbaraminto.com](https://www.barbaraminto.com/) · [Goodreads](https://www.goodreads.com/book/show/33206.The_Minto_Pyramid_Principle)).
- **Action titles / "so what" headlines** — each slide title states the takeaway as a full sentence, not a topic label; the body exists only to prove the title; the deck reads as a story from titles alone (horizontal logic) `(synthesis)` ([Slideworks](https://slideworks.io/resources/how-to-write-action-titles-like-mckinsey)).
- **Data-ink & chart discipline (Edward Tufte)** — the **data-ink ratio** from *The Visual Display of Quantitative Information* (1983; 2nd ed. 2001) ([edwardtufte.com/books](http://www.edwardtufte.com/books/)): "maximize data-ink and erase as much non-data-ink as possible" — remove redundant gridlines, decoration, "chartjunk" `(quoted via secondary scholarly source — see gaps)` ([InfoVis-Wiki](https://infovis-wiki.net/wiki/Data-Ink_Ratio)).

### 4.2 IDEO / design-consultancy report craft
The reference artifact is IDEO.org's *The Field Guide to Human-Centered Design* — a **full-colour 192-page book**, 57 methods, structured as a narrative through **Inspiration, Ideation, Implementation**, combining editorial layout, photography/case-study storytelling, and interactive worksheets ([IDEO.org Field Guide PDF](https://d1r3w4d5z5a88i.cloudfront.net/assets/guide/Field%20Guide%20to%20Human-Centered%20Design_IDEOorg_English-0f60d33bce6b870e7d80f9cc1642c8e7.pdf) · [Amazon listing](https://www.amazon.com/Field-Guide-Human-Centered-Design/dp/0991406311)). *(IDEO's specific photography/grid art direction is not documented in retrieved sources — gap.)*

### 4.3 Typographic systems (Swiss / International Typographic Style)
**Josef Müller-Brockmann**, *Grid Systems in Graphic Design* (1981) — "a cornerstone of modern design education"; the grid as a rational tool, "an aid, not a guarantee" ([Wikipedia](https://en.wikipedia.org/wiki/Josef_M%C3%BCller-Brockmann)). Swiss-style hallmarks: **asymmetric layouts on a mathematical grid, sans-serif type (Akzidenz-Grotesk, Helvetica), flush-left/ragged-right**, emphasising "simplicity, clarity, readability, and objectivity" ([International Typographic Style — Wikipedia](https://en.wikipedia.org/wiki/International_Typographic_Style)).

### 4.4 Editorial / grid principles — the load-bearing measure
Two authoritative references, cited to the correct author (do not merge):
- **Matthew Butterick, *Practical Typography*:** "Aim for an av­er­age line length of **45–90 char­ac­ters**, in­clud­ing spaces"; body text 10–12 pt print / 15–25 px web; line spacing 120–145% ([Line length](https://practicaltypography.com/line-length.html) · [Summary of key rules](https://practicaltypography.com/summary-of-key-rules.html)).
- **Robert Bringhurst, *The Elements of Typographic Style*:** "anything from **45 to 75 characters** is widely regarded as a satisfactory length of line… [the] 66-character line… regarded as ideal" ([webtypography.net 2.1.2](http://webtypography.net/2.1.2)).

Other premium baseline rules (Butterick): curly quotes, proper dashes/ellipses, one space between sentences, kerning on, bold/italic sparingly, no underline except links, "a professional font" — avoiding Times New Roman and Arial ([Summary of key rules](https://practicaltypography.com/summary-of-key-rules.html)). *(This aligns with the design-engine's banned-AI-slop-font rule.)*

### 4.5 Modern tooling for premium output
- **InDesign** — professional baseline; supports baseline + document grids and object snapping ([typst issue #1306](https://github.com/typst/typst/issues/1306)).
- **LaTeX** — publication-quality, but true baseline-grid snapping is "nearly impossible" given engine/package limits (same source).
- **Typst** — modern markup alternative aiming for LaTeX-quality output with millisecond compilation; "a publication-quality PDF with proper typography, embedded fonts, and professional layout"; drivable from Quarto ([typst.app/docs/guides/for-latex-users](https://typst.app/docs/guides/for-latex-users/) · [Quarto Typst](https://quarto.org/docs/output-formats/typst.html)).
